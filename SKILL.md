# Task Notifier — 任务通知技能

## 核心能力

当任务完成或产生重要成果时，自动通过邮件将进展和成果发送给用户。

## 使用方式

### 快速发送

直接调用脚本，无需额外参数（使用默认配置的 163 邮箱）：

```javascript
// 发送纯文本通知
node C:\Users\黄兴\.qclaw\skills\task-notifier\scripts\send_notification.cjs \
  --to henry861500x@163.com \
  --subject "任务完成：数据整理报告" \
  --body "已完成所有邮件分类统计，共处理XX封邮件..."

// 发送 HTML 精美邮件（推荐）
node C:\Users\黄兴\.qclaw\skills\task-notifier\scripts\send_notification.cjs \
  --to henry861500x@163.com \
  --subject "✅ 任务完成：数据整理报告" \
  --body "已完成所有邮件分类统计" \
  --html

// 发送带附件的通知
node C:\Users\黄兴\.qclaw\skills\task-notifier\scripts\send_notification.cjs \
  --to henry861500x@163.com \
  --subject "📊 今日数据报告" \
  --body "请查看附件中的详细数据" \
  --attach "C:\Users\黄兴\Downloads\report.xlsx" \
  --attach "C:\Users\黄兴\Documents\summary.pdf"
```

### 参数说明

| 参数 | 必填 | 说明 |
|------|------|------|
| `--to` | 否 | 收件人地址（默认发给自己） |
| `--subject` | 是 | 邮件主题 |
| `--body` | 是 | 邮件正文内容 |
| `--html` | 否 | 启用 HTML 精美格式（默认纯文本） |
| `--attach <路径>` | 否 | 附件路径，可多次使用 |
| `--config <路径>` | 否 | 自定义 .env 配置路径 |

### 自动附加成果文件

当任务产生重要文件时，必须作为附件一并发送。附件会：
- 自动检测文件大小
- 在邮件中列出文件名和大小
- HTML 邮件中以表格形式展示

### 邮件格式

**HTML 模式**（`--html`）：专业精美模板，含状态徽章、摘要卡片、附件表格
**纯文本模式**（默认）：兼容所有邮件客户端，适合简短通知

## 配置文件

邮件发送依赖 `163-email.env` 配置文件：

```
位置（按优先级）：
1. --config 参数指定路径
2. C:\Users\黄兴\.qclaw\workspace\163-email.env
3. 技能目录内的 163-email.env
```

配置文件内容：
```env
SMTP_HOST=smtp.163.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=你的邮箱@163.com
SMTP_PASS=你的授权码
```

## 邮件主题建议格式

| 任务类型 | 建议主题 |
|---------|---------|
| 任务完成 | `✅ 任务完成：<任务名>` |
| 任务失败 | `❌ 任务失败：<任务名>` |
| 定时日报 | `📅 <日期> 日报摘要` |
| 数据报告 | `📊 <主题> 数据报告` |
| 文件成果 | `📎 <任务名> - 成果文件` |

## 最佳实践

1. **重要任务必须发邮件**：所有耗时超过5分钟或有重要产出的任务，结束时都要发通知
2. **摘要要清晰**：邮件正文要包含任务核心成果，细节放附件
3. **附件要列出**：产生的文件路径在邮件正文中说明
4. **HTML 格式更专业**：正式汇报使用 `--html`，快速通知用纯文本

## 错误处理

- **找不到 .env**：提示用户先配置邮箱（`163-email.env`）
- **SMTP 连接失败**：输出具体错误信息，说明可能是授权码过期
- **附件不存在**：警告但继续发送邮件
